Les bases de données JSON

Mais avant toute chose, c’est quoi un fichier JSON ?


Selon lejournaldunet.fr : JSON (JavaScript Object Notation) est un langage léger d’échange de donnée textuelle (une base de données quoi). Il permet de représenter des données de façon structurée et optimisée.

Il est assez facile d’usage, donc bon plan pour ceux qui veulent utiliser un système de base de données en local.

Pour structurer un fichier .JSON, vous pouvez donner un nom à un tableau contenant plusieurs éléments, et dans chaque élément vous pouvez mettre autant d’item que vous le souhaitez, ou si vous ne souhaitez pas créer de tableau, vous pouvez tout simplement lister autant d’éléments que vous le souhaitez. 

            
“information”: [
                {
                "nom":"Victor",
                "age":30, 
                “pays”:”France”
                },
                {
                "nom":"Albert",
                "age":30, 
                “pays”:”France”
                }
               ]

Comment afficher les données d’un fichier JSON ?

Une fois les données stockées dans le fichier .JSON, il faudra vous rendre dans un fichier .js pour pouvoir afficher vos données. Il y a ensuite deux méthodes possibles : le JSON.parse() et le JSON.stringify()


-JSON.parse() est une fonction JS pour convertir un string en objet JavaScript. Exemple : si on parse les noms de nos tableaux informations, de cette façon-là : 

                
Const texte = JSON.parse(information)
JSON.parse(texte.nom)

voilà ce qui s’affichera : Victor, Albert



-JSON.stringify() est également une fonction JS pour convertir un string en objet JavaScript, mais d’une façon différente. En écrivant ce code : 

                    
JSON.stringify(information)

Voilà ce que nous devrions obtenir : {"nom":"Victor","age":30,“pays”:”France”};{"nom":"Albert","age":30, “pays”:”France”}

Comment on a fait ?


Tout d’abord, on s’est documentés sur le fonctionnement d’un fichier .JSON. Une fois qu’on avait bien assimilé comment tout marchait, on a commencé par créer un fichier .JSON qui allait regrouper, dans plusieurs array dédiés a chaque catégorie de personne, le nom, prénom, lien GitHub ou LinkedIn, et une photo (si disponible).

Ensuite, on a repris une navbar fonctionnelle que Louis avait fait pour un de ses projets perso, pour nous éviter tout un travail qui ne nous intéressait pas, et dans le fichier JS, on a créé une fonction qui va parse le bon array en fonction de l’adresse URL du site.

Pour faire celà, on a utilisé la fonction fetch() qui fait une requête de nos array présent dans le dossier .JSON, et lorsque la requête est dite «extraite» (c’est-à-dire qu’il y a une une réponse à la demande de requête), alors le fichier .JSON est exécuté et va parse l’array souhaité. Par exemple, si l’utilisateur clique sur la rubrique « direction », l’adresse url se terminera par ‘/TechUpdate/direction.html’. Grâce à cela, on a pu afficher les différents array dans différentes pages grâce à un fichier .JSON .
